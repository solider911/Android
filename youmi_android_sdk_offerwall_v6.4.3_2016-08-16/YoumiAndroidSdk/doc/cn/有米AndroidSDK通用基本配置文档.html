    有米 Android SDK 通用基本配置开发者文档¶

    小技巧

    欢迎您使用有米广告SDK，为了保证能正常接收有米的广告，保障各位开发者的收益，请您严格按照SDK里doc文件夹下的文档步骤来嵌入SDK ，或按照SDK里的demo源码来嵌入。

    以下注意事项，请仔细阅读：

    1. 在有米后台创建应用获取发布ID和应用密钥时，请注意区分好Android和iOS平台，不能弄错平台。
    2. 一对ID和应用密钥只能用在一个应用内，不能重复用在多个应用内，否则会拿不到正式广告。
    3. 嵌入完sdk，可以拿到测试广告，测试没问题，需上传应用给有米审核，审核通过后才能拿到正式广告，正式广告才能产生收益。
    4. 有米广告不支持模拟器，测试广告时请用真机来测试，另外，Android广告不支持Android平板，需要用手机来测试Android广告。

    一、基本配置¶

    1、导入 SDK¶

    1. 将 sdk 解压后的 libs 目录下的 YoumiSdk_*.jar 文件导入到工程指定的 libs 目录
    2. 将 sdk 解压后的 assets 目录下的 dex.jar （文件名不能变动）文件导入到工程的 assets 目录下

    小技巧

    PS :

    你可能注意到sdk解压后还存在一个 extralibs 目录，该目录存放我们和其他第三方厂商合作所需要用到的扩展jar，属于 可选导入的jar ，只有你用到某些功能的时候，才需要导入，在你使用该功能的时候，该功能的文档会说明需要引入哪一个jar

    e.g :

    • libsmmsdk.jar : 微信的分享sdk，在分享任务墙中需要引入
    • miadtracker.jar : 小米游戏中心渠道统计sdk，当你的 应用/游戏需要发布到小米游戏中心 的时候，还需要导入本jar到你的工程项目的 libs 目录中

    2、权限配置¶

    请将下面权限配置代码复制到 AndroidManifest.xml 文件中：

    <uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.READ_PHONE_STATE" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
<uses-permission android:name="android.permission.ACCESS_WIFI_STATE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.SYSTEM_ALERT_WINDOW" />

<!-- 如果使用积分墙广告,还需要配置下面权限 -->
<uses-permission android:name="android.permission.GET_TASKS" />
<uses-permission
            android:name="android.permission.PACKAGE_USAGE_STATS"
            tools:ignore="ProtectedPermissions" />

<!-- 以下为可选权限 -->
<uses-permission android:name="com.android.launcher.permission.INSTALL_SHORTCUT" />

    小技巧

    当配置权限 android.permission.PACKAGE_USAGE_STATS 时，还需要在 manifest 标签中配置下面内容 xmlns:tools="http://schemas.android.com/tools"

    3、初始化应用信息¶

    请务必在应用第一个 Activity（启动的第一个类）的 onCreate 中调用以下代码

    import net.youmi.android.AdManager;
...
AdManager.getInstance(Context context).init(String appId, String appSecret, boolean isTestModel, boolean isEnableYoumiLog);

    小技巧

    参数说明：

    • appId 和 appSecret 分别为应用的发布 ID 和密钥，由有米后台自动生成，通过在有米后台 > 应用详细信息 可以获得。
    • isTestModel : 是否开启测试模式，true 为是，false 为否。（上传有米审核及发布到市场版本，请设置为 false）
    • isEnableYoumiLog: 是否开启有米的Log输出，默认为开启状态
      • 上传到有米主站进行审核时，务必开启有米的Log，这样才能保证通过审核
      • 开发者发布apk到各大市场的时候，强烈建议关闭有米的Log

    4、混淆配置¶

    在积分墙v6.3.0之后，有米的SDK支持二次混淆了，开发者不用再为有米SDK填写混淆配置了，但是为了您的应用安全，强烈建议贵应用开启混淆打包功能

    5、编译打包（重要）¶

    危险

    注意： 打包时，如果您在AndroidManifest.xml文件中指定了android:targetSdkVersion并且targetSdkVersion>=17，
    那么需要在配置文件project.properties中指定 target-sdk 为17以上。 查看为何使用 target-sdk 17 编译

    6、设置渠道号（可选）¶

    目前，有米已经与多家应用商店合作，当您发布的目标应用商店有要求您加上有米的渠道号时，您可以按以下操作加入渠道号进行打包，这样可以保证您的应用在该应用商店上顺利审核通过。

    渠道号列表详见 有米广告推广渠道号列表

    在AndroidManifest.xml中添加以下代码:

    <meta-data
    android:name="YOUMI_CHANNEL"
    android:value="这里替换为非负整数的渠道号" >
</meta-data>

    6.1 小米游戏商店¶

    有米跟小米游戏商店达成合作，如果你的应用/游戏需要上传发布到小米游戏商店，那么只需要做额外一点工作，就可以保证您的应用能顺利通过小米游戏商店的审核。

    1. 在 AndroidManifest.xml中填写小米游戏中心的渠道号 : 11030
    2. 在你的项目中 libs 目录下导入添加我们sdk解压后目录中的小米游戏商店的统计sdk(extralibs/miadtracker.jar)
    3. 最后，加入该sdk的混淆配置：
    -keep public class com.mi.adtracker.MiAdTracker{ *; }

    小技巧

    至此，恭喜您已经准备好有米Android SDK的通用基本配置。
